## 1. プロジェクト概要

岩手県平泉町と連携した地域活性化マッチングプラットフォーム。
**ホスト（企業）** が「お手伝い募集」を投稿し、**ひらいず民（ワーカー）** が応募・マッチングして地域活動に参加する。

---

## 2. アクター定義

本システムには **4種類のアクター** が存在する。

```
┌─────────────────────────────────────────────────────────────────┐
│                       ちほプロジェクト                            │
│                                                                 │
│  ┌──────────────┐ ┌──────────────┐ ┌──────────┐ ┌────────────┐  │
│  │  ひらいず民   │ │    ホスト     │ │  管理者   │ │    役場    │  │
│  │   (worker)   │ │  (company)   │ │  (admin) │ │ (municipal)│  │
│  │              │ │              │ │          │ │            │  │
│  │ 個人の応募者 │ │ 平泉町の企業 │ │ システム │ │ 平泉町役場 │  │
│  │              │ │              │ │ 運営者   │ │ の職員     │  │
│  └──────┬───────┘ └──────┬───────┘ └────┬─────┘ └─────┬──────┘  │
│         │                │               │             │         │
│    募集に応募       募集を投稿       全体を管理     統計を閲覧   │
│    チャット         応募を管理       ユーザー管理   CSV出力     │
│                    チャット         操作ログ       操作ログ    │
└─────────────────────────────────────────────────────────────────┘
```

| # | アクター名 | role値 | 認証ガード | アクセスURL | 説明 |
|---|---|---|---|---|---|
| 1 | **ひらいず民** | `worker` | web | `/` | 募集を検索し応募する個人ユーザー |
| 2 | **ホスト** | `company` | web | `/` | 募集を投稿・管理する平泉町の事業者 |
| 3 | **管理者** | `admin` | admin | `/admin` | システム全体を管理する運営者 |
| 4 | **役場** | `municipal` | municipal | `/municipal` | 統計閲覧・CSV出力する平泉町役場職員 |

### 補足: 5番目のアクター？

| 状態 | 説明 |
|---|---|
| **未認証ユーザー（ゲスト）** | ログインしていない訪問者。トップページと募集一覧・詳細のみ閲覧可能。アクターとして定義するかは運用方針次第。 |

## 3. アクター別 できること／できないこと

### 3.1 ひらいず民（worker）

| できること | できないこと |
|---|---|
| ✅ 会員登録（メール＋パスワード） | ❌ 募集の投稿・編集・削除 |
| ✅ ひらいず民プロフィール登録・編集 | ❌ 応募の承認・却下 |
| ✅ 募集一覧の閲覧・検索（タグ絞り込み） | ❌ 他ユーザーのプロフィール編集 |
| ✅ 募集詳細の閲覧（地図表示） | ❌ 管理パネル (`/admin`) へのアクセス |
| ✅ 募集への応募（動機・理由） | ❌ 役場パネル (`/municipal`) へのアクセス |
| ✅ 自分の応募履歴の閲覧 | ❌ CSV出力 |
| ✅ ホストとのチャット（応募後） | ❌ ユーザー管理・操作ログ閲覧 |
| ✅ パスワード変更・2要素認証設定 | ❌ 同じ募集への重複応募（DB制約） |
| ✅ アイコン画像のアップロード | |

### 3.2 ホスト（company）

| できること | できないこと |
|---|---|
| ✅ 会員登録（メール＋パスワード） | ❌ 募集への応募 |
| ✅ ホストプロフィール登録・編集 | ❌ 他社の募集の編集・削除 |
| ✅ 自社募集の削除（応募がない場合のみ） | ❌ 応募がある募集の削除（FK制約: restrictOnDelete） |
| ✅ 募集の投稿・編集・削除（応募なしのみ） | ❌ 管理パネル (`/admin`) へのアクセス |
| ✅ 自社募集一覧（マイ募集）の管理 | ❌ 役場パネル (`/municipal`) へのアクセス |
| ✅ 自社募集への応募一覧の閲覧 | ❌ CSV出力 |
| ✅ 応募の承認（accepted）・却下（rejected） | ❌ ユーザー管理・操作ログ閲覧 |
| ✅ ひらいず民とのチャット（応募後） | ❌ 他社募集への応募管理 |
| ✅ パスワード変更・2要素認証設定 | |
| ✅ アイコン画像のアップロード | |

### 3.3 管理者（admin）

| できること | できないこと |
|---|---|
| ✅ Filament管理パネル (`/admin`) へのアクセス | ❌ ユーザー側の募集投稿・応募（UIなし） |
| ✅ 全ユーザーの一覧・作成・編集・削除 | ❌ ユーザー側のチャット（UIなし） |
| ✅ 全応募の一覧・詳細・ステータス変更 | |
| ✅ 応募データのCSVダウンロード | |
| ✅ 操作ログ（Activity Log）の閲覧 | |
| ✅ ロール・権限の管理（Filament Shield） | |
| ✅ `super_admin` は全ポリシーをバイパス | |

### 3.4 役場（municipal）

| できること | できないこと |
|---|---|
| ✅ Filament役場パネル (`/municipal`) へのアクセス | ❌ ユーザーの作成・編集・削除 |
| ✅ 全応募の一覧・詳細の **閲覧**（読取専用） | ❌ 応募ステータスの変更 |
| ✅ 応募データのCSVダウンロード | ❌ ロール・権限の管理 |
| ✅ 操作ログの **閲覧**（読取専用） | ❌ ユーザー側機能の操作 |
| | ❌ 管理パネル (`/admin`) へのアクセス |

## 10. 漏れ・課題の指摘

### 10.1 アクターについて

現在の4アクター（ひらいず民、ホスト、管理者、役場）で **主要アクターは網羅されている**。

ただし以下を検討すべき：

| 検討事項 | 内容 | 対応案 |
|---|---|---|
| **未認証ユーザー（ゲスト）** | 現状トップ・募集一覧・詳細は閲覧可能。5番目のアクターとして明示すべきか | 仕様書に「閲覧のみ可能」と明記する（本書では記載済み） |
| **プロフィール未登録ワーカー** | 登録直後でプロフィール未作成の状態。`role:worker` ミドルウェアで弾かれる画面がある | 現状は登録画面にリダイレクトされる。問題なし |
